### [สรุปงาน] ระบบ Map และการจัดการ Hazard Layer

พวกเราทำระบบแผนที่โดยใช้ MapLibre เพื่อแสดงข้อมูลภัยพิบัติแบบเรียลไทม์ 
ทั้งแบบที่เป็นรูปภาพ (Raster) และแบบที่เป็นข้อมูลพิกัด (Vector) ครับ

1. การอัปเดตเลเยอร์ (Dynamic Updates): 
ทุกครั้งที่ User เปลี่ยนประเภทภัยพิบัติหรือช่วงเวลา เราต้องสั่งลบ Layer และ Source 
อันเก่าทิ้งก่อนแล้วค่อยวาดอันใหม่ทับลงไป จุดนี้ต้องระวังเรื่อง Performance 
ถ้า User กดเปลี่ยนรัวๆ อาจจะทำให้แผนที่ "วูบ" (Flicker) หรือแอปค้างได้ครับ

2. ความต่างของ Vector และ Raster: 
สองโหมดนี้ลอจิกข้างหลังต่างกันเยอะมาก ทั้งวิธีกำหนดสีและวิธีดึงข้อมูล 
เราเลยต้องทำระบบ Standardize ไว้ที่ Service เพื่อให้หน้าบ้านไม่ต้องวุ่นวาย 
แค่บอกโหมดมา เดี๋ยวระบบจัดการเรื่องพ่นสี (Styles) ให้เองอัตโนมัติ

3. ข้อมูลชุดใหญ่ (Large Datasets): 
เมื่อข้อมูลมีจำนวนเยอะมากๆ เราต้องคิดเรื่องการทำ Tiling หรือลดความละเอียดลง 
เพื่อให้แผนที่ยังลื่นอยู่ ไม่งั้นเครื่อง User จะแบกแรมไม่ไหวและทำให้เบราว์เซอร์อืดครับ

ไฟล์สำคัญที่ต้องดู:
- hooks/useHazardLayer.ts: ตัวรับหน้าที่วาดและลบเลเยอร์บนแผนที่ตามที่ User เลือก
- services/map.service.ts: ตัวรับจบเรื่องการคำนวณ URL และการเลือกสี (Styles) 
- components/MapLibre.tsx: ตัวโครงสร้างแผนที่หลักและปุ่มควบคุมต่างๆ
