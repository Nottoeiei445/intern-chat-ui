### [สรุปงาน] ระบบ Auth การจัดการ Token และ Session
พวกเราทำระบบ Auth ที่จัดการครบวงจร ทั้งตอน Login, Logout และการแอบไปขอตั๋วใหม่ (Token Refresh) 
แบบที่ User ไม่รู้ตัวเพื่อให้เล่นเว็บได้ต่อเนื่องยาว ๆ 
1. การขอ Token พร้อมกัน (Concurrent Refresh): ถ้าเรายิง API 5 อย่างพร้อมกันตอนตั๋วหมดพอดี ถ้าไม่ดักไว้ แอปจะวิ่งไปขอตั๋วใหม่ 5 รอบซ้อนจนหลังบ้านงง เราเลยใช้ตัวแปร `isRefreshing` มาล็อคไว้ แล้วให้คนอื่นต่อคิวรอ
ใน `refreshSubscribers` พอได้ตั๋วมาปุ๊บ ตัวแรกจะเอาตั๋วมาให้คนในคิวแล้วค่อยปล่อยทุกคนวิ่งไปทำงานต่อครับ

2. เส้นแบ่ง Server/Client (SSR Boundary): 
`localStorage` เนี่ยมันมีแค่ใน Browser ครับ แต่ Next.js บางทีมันแอบรันโค้ดที่ฝั่ง Server ด้วย 
ถ้าเราเผลอไปสั่ง `getItem` ตอนมันรันที่ Server แอปจะระเบิด (Runtime Error) ทันที 
ต้องคอยเช็ค `typeof window !== "undefined"` ให้ดีก่อนใช้ครับ

3. Cleanup: 
เรามีตัวจับเวลาแอบเช็คตั๋วอยู่เรื่อย ๆ (Intervals) ถ้า User เค้า Logout ออกไปแล้ว 
เราต้องสั่งหยุดไอ้ตัวจับเวลานี้ด้วย ไม่งั้นมันจะทำงานค้างอยู่ในเครื่องเปล่า ๆ 

ไฟล์สำคัญที่ต้องดู:
- config/auth.config.ts: รวมทุกค่าไว้ เปลี่ยน URL หรือตั้งเวลา หมดอายุ
- context/AuthContext.tsx: ตัวคุมสถานะทั้งแอปว่าใครอยู่ใครไป
- services/auth.service.ts: ตัวใช้ Axios ยิงหาหลังบ้าน มี Interceptor คอยดัก Error 401 
  เพื่อวิ่งไปขอตั๋วใหม่ให้เราอัตโนมัติครับ
