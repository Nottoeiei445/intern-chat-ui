### [สรุปงาน] ระบบ Chat การจัดการข้อความและ AI Integration (Update: Streaming & Guest Mode)

พวกเราอัปเกรดระบบแชทให้รองรับการตอบกลับแบบเรียลไทม์ และเพิ่มโหมดเข้าใช้งานชั่วคราว 
รวมถึงจัดการปัญหาเรื่อง Endpoint ไม่ตรงกับหลังบ้านครับ

1. การรับส่งข้อความแบบ Streaming (SSE):
ตอนนี้เปลี่ยนมาใช้ท่า Stream (SSE) โดยใช้ ReadableStream และ TextDecoder เพื่อแกะข้อมูล
ที่ไหลมาทีละตัวหนังสือ และเพิ่มลอจิกตัดแท็ก <tool_call> ออก เพื่อให้แสดงผลเฉพาะเนื้อหาที่
ต้องการโชว์บน UI จริงๆ ช่วยให้ User ไม่ต้องรอนานเหมือนตอนรับ JSON ก้อนเดียวครับ

2. โหมดขาจรและการจัดการ State (Guest Mode):
เพิ่มการแยก State ระหว่างแชทปกติ (chats) กับแชทชั่วคราว (ephemeralMessages) โดยถ้า User 
ยังไม่ได้ล็อกอิน ระบบจะสลับไปใช้โหมด Ephemeral อัตโนมัติ ข้อมูลจะถูกเก็บไว้แค่หน้าบ้าน 
ไม่ส่งไปบันทึกลงฐานข้อมูล เพื่อให้คนทั่วไปลองเล่นได้โดยไม่ต้องสมัครสมาชิก

3. การ Mapping ไอดีจริงและแก้ไข Endpoint:
แก้ไขปัญหา Path การลบแชท (/chat/delete) ให้ตรงกับหลังบ้าน และเพิ่มลอจิกดักจับไอดีจริงจาก 
Response Header เพื่อเอามาทับไอดีชั่วคราว (session_xxx) ทันทีที่เริ่มการสนทนา 
ทำให้ระบบจัดการประวัติแชทและฟังก์ชันลบทำงานได้ถูกต้องไม่ติด Error 404/500 ครับ

ไฟล์สำคัญที่ต้องดู:
- lib/api-client.ts: เพิ่ม requestRaw และ stream method สำหรับรองรับข้อมูลแบบไหล
- hooks/useChat.ts: ตัวคุมลอจิก Stream, การ Parse ข้อมูล SSE และการแยกโหมด Guest
- config/chat.config.ts: แก้ไข Path Endpoint (เช่น delete) ให้ตรงกับ Backend
- features/chat/index.tsx: ลอจิกการตัดสินใจเลือกโหมดส่งข้อความ (Ephemeral vs Member)
